# Security Policy

## Reportar vulnerabilidades

No abras vulnerabilidades sensibles en issues publicos.

Canal recomendado:

- Contacto inicial via mantenedores del repositorio.
- Incluye pasos de reproduccion, impacto y version afectada.

## Buenas practicas locales

- Usa dependencias oficiales y actualizadas.
- Revisa plugins antes de habilitarlos.
- Mantiene `config.json` y certificados fuera de repositorios publicos cuando corresponda.
